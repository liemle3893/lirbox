What this write-up covers Three improvements to the conductor skill's generator — all surgical edits to one .cjs file plus its eval net and docs. By the end you should know exactly which combos change, which stay byte-for-byte identical, and the two invariants that keep that promise honest.

TL;DR

Cheaper model tiers, and delivery PRs that finally carry a write-up + a design diagram.

The conductor generator gains --model-mode balanced (haiku for mechanical work, opus for thinking, sonnet for the work phases) and a new Writeup phase that promotes the per-worker implementation-notes and generates a writeup.html + design diagram, committed under docs/changes/<name>/ so they ride the PR. The headline guarantee: default mode is byte-for-byte identical to before — every new opt is emitted only when you ask for it.

Why

Three gaps in the delivery harness

conductor generated a clean workflow, but threw away its best review material and ran every worker on one model. Per-worker implementation-notes were folded into a markdown summary and then dropped; the PR phase pushed a bare TODO body; and a five-line config edit and a hard reasoning gate both ran on the same expensive model.

Before

  • implementation-notes/*.html consumed by DocsGate, then gitignored and discarded — the reviewer never saw them.
  • PR phase = git push + gh pr create with a literal "TODO title"/"TODO summary".
  • No notion of model per task — every checkpoint, push, and review ran on the session model.

After

  • A Writeup phase promotes the notes into docs/changes/<name>/notes/ and adds a writeup.html + design.html, all committed.
  • The PR body links those artifacts; DocsGate's summary.md joins them in the same per-run dir.
  • --model-mode balanced tiers each phase by class — haiku / opus / sonnet — opt-in and overridable.

Where to focus your review

The parts that actually need eyes

The byte-identity invariant

Every new fragment must be emitted only when opted in. The check that proves it: regenerate the full matrix from the pre-change generator and from this one with --no-writeup in default mode — the only diff is the intentional DocsGate summary.md path. Worth confirming the composition in agentCall/gateLoop can never leak a trailing space or stray opt.

scaffold-workflow.cjs · agentCall(), gateLoop(), mechFrag

2

Phase → model-class assignment

Each descriptor passes mdl('mechanical'|'think'|'work'). The judgement calls: Brief and Verify/ReVerify. Brief is "think" (it derives the goal that gates everything); Verify/ReVerify are "mechanical" (they run a known suite and report). If you'd tier them differently, it's a one-word edit.

scaffold-workflow.cjs · PHASES table

3

Writeup hard-fails before PR

The Writeup phase throws if it can't write artifacts, and it sits before PR — so "every PR carries reviewer artifacts" is actually enforced (no artifacts ⇒ no PR). The escape hatch is --no-writeup. Confirm that ordering is the behaviour you want for small/lite PRs.

scaffold-workflow.cjs · Writeup descriptor

4

Cross-repo skill invocation

The Writeup worker invokes lirbox:pr-writeup and lirbox:flowchart by skill name (portable — conductor runs on any repo, but the sibling skills live in the plugin, not the target tree), with a read-the-SKILL.md fallback. This is a runtime behaviour the eval net can't fully exercise; the assertion only checks the prompt wiring.
