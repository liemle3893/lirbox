Re-read the spec (docs/superpowers/specs/2026-06-25-prospector-design.md) and audited the implementation section by section (config §1, loop+ledger §2, two-clock budget §3, isolation+safety §4, reporting §5, overnight trigger §6, components, success criteria). Verified node --check on all four .cjs and ran test-optimize.cjs (41 assertions, exit 0). Found and fixed four real bugs and reconciled the docs to match. None were cosmetic — each was a correctness or safety gap.

BUG 1 (safety, spec §4 / criterion 5): surface lock missed new untracked files

The eval worker reported diffFiles = git diff --name-only, which lists ONLY tracked unstaged modifications and does not show new untracked files. An experiment that creates a brand-new out-of-surface file (e.g. a new benchmark harness, or a stray top-level script) would produce an empty/short diff, pass surfaceAllows, and slip the anti-metric-gaming fence — directly defeating spec §4 and success criterion 5. Fixed the eval prompt to list every changed path with git -c core.quotepath=false status --porcelain --untracked-files=all (reporting the post-rename path for renames) and reconciled the three doc descriptions (loop-runtime §4, metric-gate §7, SKILL gotcha) that said bare git diff --name-only. Added a test marker asserting the untracked-aware command is emitted.

BUG 2 (safety, spec §4 / criterion 2 & 8): discard reverted only the surface

On DISCARD the revert worker ran git checkout -- $SURFACE + git clean -fd -- $SURFACE, both scoped to the surface. But one of the discard reasons is "touched files OUTSIDE the surface" — and a surface-scoped revert leaves those out-of-surface edits/new-files in the worktree, polluting the next experiment and potentially getting swept into a later KEEP commit. Spec §4 requires git status clean after every discard. Changed the revert to reset the WHOLE worktree to HEAD (git reset --hard HEAD + git clean -fd); prior KEPT commits are already on the branch so only the uncommitted candidate is dropped. Reconciled the loop-runtime §4 and SKILL gotcha descriptions and added test markers for both commands.

BUG 3 (correctness, multi-glob surface): raw $SURFACE interpolation broke staging/reverting

surfaceAllows splits the surface on [,\n] (so a comma/newline-separated multi-glob surface is supported for the LOCK), but the keep used git add -- $SURFACE, interpolating the raw string. For a surface like src/a/**,src/b/** that yields git add -- src/a/**,src/b/**, which git treats as one pathspec with a literal comma — wrong. Since the surface lock already guarantees every changed path is ⊆ the surface before a KEEP, the staging is now git add -A (safe by construction, no fragile globbing). BUG 2's full-worktree reset removed the same raw-$SURFACE problem from the revert path.

BUG 4 (correctness, spec §5 / criterion 7): resume dropped the persisted baseline

The checkpoint payload writes baseline: results.baseline || null as a whole-object overwrite (the startedAt-preserving merge only protects startedAt/finishedAt). On resume the conductor never restored results.baseline from args, so the first post-resume checkpoint nulled out the saved baseline block — and optimize-report's baseline→best % improvement (criterion 9) reads state.baseline, so a resumed run would report n/a. Fixed by re-passing baseline in the resume args (SKILL step 4 + loop-runtime §5) and restoring results.baseline = args.baseline in the conductor; the EVALCAP-on-resume branch now derives the cap from the real measured baseline.evalSec instead of falling to the floor. The original code even left a comment anticipating this ("results.baseline is not carried in args").

DOC FIX (no behavior change): scaffold invocation flag was wrong in SKILL step 2

SKILL.md told the user to run scaffold-optimize.cjs --config .optimize/config/<name>.json, but the generator's CLI only accepts --name <slug> (config is data-in, never baked). That invocation would have failed with "--name is required". Corrected to --name <name> and clarified that config is passed at launch via Workflow args.config, not baked.
