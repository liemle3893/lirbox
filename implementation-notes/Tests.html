test-optimize.cjs — implementation notes

Decision: "representative configs" became representative slugs

Conductor's test-scaffold.cjs varies flags/profiles because conductor bakes the phase sequence into the template. Prospector's scaffold-optimize.cjs deliberately does NOT bake the run config — goal/surface/metric/gate/budgets are passed in via Workflow args.config at launch — so the emitted loop body is BYTE-IDENTICAL for every run except the baked-in slug (meta.name / NAME). The generator also takes only --name/--out/--force, no --config. So the "matrix" varies the slug (and exercises the kebab validator's edges: hyphenated, numeric, single-char) and the structural assertions do the real work. This is an intentional deviation from conductor's flag-matrix shape, forced by the data-in design.

Decision: isBetter boundary is >= minDelta, so equal ties through at minDelta=0

My first draft asserted that an EQUAL metric with minDelta=0 should NOT keep (strict improvement). The harness caught this against the real implementation: isBetter is documented and coded as "beat best by AT LEAST minDelta", i.e. metric - best >= d. With the default minDelta=0 that is 0 >= 0 = true, so an equal metric is a (harmless) keep; strict improvement requires a positive minDelta. I changed the TEST to pin this documented boundary rather than touch the generator — the generator is the source of truth and the >= semantics match the spec's "ignore moves BELOW minDelta" (a tie is not below). Reviewer confirm: equal-metric keeps at minDelta=0 are acceptable (they only ever replace best with an equal-value commit; monotonicity in criterion 3 holds with <=/>=, not strict).

Decision: a string-scan "no-fs guard" beyond node --check

node --check only proves the emitted loop parses; it cannot prove the conductor layer obeys the restricted-layer rule (no fs/require/Date.now/ Math.random/new Date — every side-effect must live inside an agent() worker prompt string). I added Gate D: strip the agent(`…`) template literals (worker instructions are data, not executed conductor code), then forbid the restricted primitives in what remains. This is the prospector analog of conductor's "meta. at runtime" guard — a class of bug node --check is blind to. Caveat: the template-literal stripper is a regex (/`(?:[^`\\]|\\.)*`/g); it is correct for the currently-generated prompts (no nested unescaped backticks at the top level) but would need revisiting if the generator ever emits nested template literals inside a worker prompt.

Cross-check of the spec's 9 Verifiable success criteria

What this build satisfies now (static / unit level) vs. what needs a LIVE optimization run.

#CriterionCoverage in this build
1Config derivation (metric/parse/direction/gate run; halts for confirm) LIVE — out of scope. Requires a real repo + the §1b auto-propose agent + AskUserQuestion. No static surrogate; not the generator's job.
2Keep rule: KEPT iff gate passed AND metric beat best by ≥ minDelta; else discard + clean worktree Partial (decision half, statically). The metric half is unit-tested via isBetter (both directions, minDelta boundary, non-finite). Structure asserts the keep = gatePassed && beats && surfaceOk conjunction and the discard git checkout -- $SURFACE revert exist. The actual gate/commit/clean behavior is LIVE (worker side-effects).
3Monotonic best; checking out opt/<name> reproduces best.metric LIVE — out of scope. Monotonicity follows from the keep rule (best only advances when isBetter), which is unit-tested; reproducing the metric on a real checkout needs a live run.
4Per-experiment bound: propose ≤ agentCapSec, eval ≤ evalCapSec; timeout ⇒ discard Partial. deriveEvalCap (≈3×, floor, override, junk input) is unit-tested. Structure asserts the propose/eval workers and the EVALCAP/AGENTCAP wiring exist. Enforcing the actual timeout is a LIVE worker behavior.
5Surface lock: out-of-surface edit is discarded Partial. Structure asserts both surfaceAllows(diffFiles, SURFACE) call and the surfaceAllows pure-glob helper exist. The helper itself is pure JS and could be unit-tested directly, but the generator does NOT export it (it is emitted as a string), so a direct unit test isn't wired here. End-to-end "inject an out-of-surface edit ⇒ discard" is LIVE.
6Stop conditions: first of {experiments, wallclock, tokens, plateau}; status reflects why Satisfied at unit level. shouldStop is unit-tested on each budget, plateau on/off, and precedence (experiments first). Structure asserts the in-loop shouldStop( call and the stopReason in finalize.
7Resume: re-invoke continues from ledger, KEPT commits intact, no repeats Partial. Structure asserts the resume reachability guard (Unreachable resume:). Full resume (ledger replay, no repeated experiments) is LIVE.
8Non-destructive: main byte-unchanged; no auto-merge LIVE — out of scope. A property of worktree isolation + the absence of any merge step; verifiable only by a real run.
9Report: optimize-report.cjs outputs baseline→best %, runs/kept, duration, tokens, cost Out of scope for THIS harness (covered by node --check on optimize-report.cjs here; a report-content test belongs to that script's own net, not the generator net).

Summary: criteria 6 is satisfiable at unit level and is; 2/4/5/7 are partially satisfied statically (their pure decision halves are unit-tested, their structure markers asserted) but their side-effecting halves require a LIVE run. Criteria 1, 3, 8 require a LIVE optimization run (real repo + real metric + real worktree/git) and are therefore out of scope for this generator regression net. Criterion 9 is exercised only via node --check here.
