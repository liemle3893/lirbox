prospector skill — implementation summary

plugins/lirbox/skills/prospector/ implements the approved design (docs/superpowers/specs/2026-06-25-prospector-design.md): a sequential keep-or-discard hill-climbing loop (autoresearch-style "generations") that auto-proposes a metric+gate, confirms once, then optimizes one declared surface behind a hard gate on an isolated, never-auto-merged branch — built on conductor's durable, resumable, worktree-isolated Workflow backbone.

What was built (and where it maps in the spec)

  • SKILL.md — frontmatter + $ARGUMENTS one-arg auto-detect (list / resume / new goal), procedure §1→§5, the §1b auto-propose+confirm+measure-baseline step, gotchas, bundled-resources list. Mirrors conductor's conventions.
  • references/metric-gate.md — §1 config shape, deriving metric cmd/parse/direction and the hard gate, the DECLINE rule (§1 anti-pattern guard), baseline measurement (§1/§3), surface lock (§4), and a ready-to-use setup-agent prompt + schema.
  • references/loop-runtime.md — §2 two-layer conductor/worker model, ledger schema, keep-or-discard + surface-lock + two-clock-budget rules (§2/§3/§4), resume protocol, common mistakes.
  • scripts/scaffold-optimize.cjs — SoT generator for the loop conductor (.optimize/<name>.js): emits Setup (worktree from fresh origin, §4) → Baseline (gate-must-pass → metric → measure evalSec → derive cap, §1/§3) → bounded experiment loop (propose → bounded eval → conductor keep/discard with surface-lock → checkpoint, §2/§3/§4) → finalize. Config is data-in via args.config (resume re-passes it); the loop body is pure JS — no fs/git/Date.now/Math.random (all side-effects in agent() workers). Exports the pure helpers isBetter/shouldStop/deriveEvalCap.
  • scripts/list-optimizations.cjs — list runs from .optimize/state/ (in-progress by default; --all).
  • scripts/optimize-report.cjs — §5 report: baseline→best (signed % improvement by direction), runs/kept, plateau, duration, tokens, est cost → .optimize/reports/<name>.md. Criterion 9.
  • scripts/test-optimize.cjs — regression net: generates a slug matrix, node --checks each emitted loop, asserts the loop structure + the no-fs conductor-layer guard, and unit-tests the pure helpers.

Run-state namespace (mirrors conductor's .workflows/)

.optimize/config/<name>.json (approved config, §1), state/<name>.json (durable ledger + run state, §2), <name>.js (generated conductor), reports/<name>.md (§5) — all in the MAIN repo so they survive worktree removal; code edits live on branch opt/<name> in .worktrees/opt-<name>.
